# 프로젝트 목적과 구현 범위

작성 기준일: 2026-06-28

## 최종 목적

이 프로젝트의 최종 목적은 KOSPI와 NASDAQ을 대상으로 한 주식 자동 매매 시스템을 개발하는 것이다.

단, 현재 구현 상태는 자동 주문 시스템이 아니라 자동매매를 위한 판단 기반을 만드는 단계다. 현재 시스템은 시장 상황, 섹터 모멘텀, 주도주 후보, 리스크, 행동 가이드의 일부를 계산하고 보여주는 리서치/의사결정 보조 도구에 가깝다.

## 현재 위치

```text
최종 목표: 자동매매 시스템
현재 단계: 시장/섹터/종목 판단과 전략 수립 기반 구축
아직 아님: 검증된 매매 신호, 자동 주문, 브로커 연동, 수익률 보장
```

## 대상 시장

| 시장 | 범위 |
|---|---|
| KOSPI | 한국 증시, KRX 기반 지수/수급/섹터 정보 일부 |
| NASDAQ | 미국 성장주 중심 증시, Yahoo/FRED 등 기반 지수/섹터/종목 정보 일부 |

## 시스템이 답해야 하는 질문

1. 지금 전체 증시는 위험 선호 환경인가, 방어해야 하는 환경인가?
2. KOSPI와 NASDAQ 중 어디에 더 큰 비중을 둘 것인가?
3. 현금, 본주, 2배 레버리지의 비중은 어떻게 둘 것인가?
4. 어떤 섹터가 현재 주도하고 있고, 어떤 섹터가 다음 후보인가?
5. 섹터 내 Key Player와 Rising Star는 누구인가?
6. 연준, 정부, 지표 발표, 산업/기업 이벤트가 판단을 바꾸는가?
7. 현재 보유 포트폴리오를 어떻게 리밸런싱할 것인가?
8. 개별 종목/ETF의 진입, 손절, 비중, 체크아웃 조건은 무엇인가?
9. 매매 결과를 어떻게 모니터링하고 규칙으로 개선할 것인가?

## 구현 상태 표기

모든 단계와 모듈은 아래 상태 중 하나 이상으로 표시한다.

| 상태 | 의미 |
|---|---|
| Planned | 기획만 있음 |
| Partial | 일부 구현됨 |
| Implemented | 기능은 동작함 |
| Tested | 테스트 있음 |
| Validated | 과거 데이터/실전 검증 통과 |
| Operational | 배포 또는 정기 갱신 중 |

## 중요한 경계

- 자동매매가 최종 목표이지만, 검증 전 판단은 확률이나 승률로 표현하지 않는다.
- 현재 산출은 투자 조언이 아니라 리서치/의사결정 보조 신호다.
- 주문 실행, 브로커 API, 개인화 투자 권유는 현재 구현 범위 밖이다.
- 검증되지 않은 룰은 `unvalidated`, `verified=false`, 또는 이에 준하는 표시를 유지해야 한다.
