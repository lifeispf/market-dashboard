# 유동성 천장 모니터링 대시보드 — 확장 스펙 §14 (전환·동반붕괴 사전감지 레이어)

> 이 문서는 v2 기획안(`기획안.md`)과 §13 확장 스펙(`기획안-v3-확장스펙.md`)에 이어지는 **세 번째 분책(§14)** 이다.
> v2는 "천장이 어디인가(여력)", §13은 "환경이 어느 쪽으로 기울어 있나(방향성)"를 봤다. §14는 한 발 더
> 나아가 **"방향이 곧 바뀌려 하는가 — 특히 상승에서 보합·하락으로의 전환, 그리고 순환매가 무너지며
> 일시에 동반붕괴하는 국면을 가격보다 먼저(또는 즉시) 잡는가"** 를 본다.
> **v2 본문과 §13을 먼저 읽은 사람을 전제**로 쓴다. 용어·데이터소스 표기·config 분리·하드 러슨 컨벤션은
> v2/§13을 그대로 따른다.

## 14-0. 설계 철학 — 세 개의 냉정한 전제

전환 탐지는 v2/§13의 다른 어떤 레이어보다 **오해와 오탐의 위험이 크다.** 구현 전에 세 전제를 못 박는다.

### 전제 1 — "가격에서 파생된 지표는 전환을 선행하지 못한다"

v2/§13의 신호 대부분(공탐·밴드·이격도·로테이션)은 가격에서 계산된다 → 가격과 동행하거나 후행한다.
전환을 **하루라도 미리** 잡으려면 **"가격은 아직 신고가인데 그 밑에서 먼저 갈라지는 것"** 을 봐야 한다.
§14의 모든 지표는 이 기준으로 선별됐다: ① 옵션시장 구조(VIX 기간구조·내재상관), ② 시장 내부 폭
(breadth 다이버전스), ③ 크로스에셋 위험회피, ④ 모멘텀의 2차 미분(가속도 둔화).

### 전제 2 — "단일 지표 경보는 반드시 실패한다 → confluence(합의)로만 간다"

일별 데이터로 일 단위 전환을 잡는 건 본질적으로 노이즈와의 싸움이다. **어떤 단일 지표도 오탐이
적중보다 압도적으로 많다.** §14는 개별 지표로 "전환이다"를 외치지 않는다. 4~5개 약한 선행신호를
각각 −2~+2로 점수화 → 합산 → **"3개 이상 동시 점등"일 때만 경보**를 울리는 confluence 게이지로 간다.
그리고 **반드시 백테스트로 "경보 점등 후 N일 수익률 분포"를 검증**해 선행성을 숫자로 확인하고
가중치를 잡는다(§14-7-G). 검증 없이 그럴듯한 지표만 모으면 작동 안 하는 장식이 된다.

### 전제 3 — KOSPI는 위성시장 → **글로벌(미국) 지표를 프록시로 차용**한다

이것이 §14의 가장 중요한 구조적 결정이다. KOSPI는 글로벌, 특히 미국 증시에 **후행하는 위성시장**이다.
전환·붕괴 선행신호의 핵심인 VIX 기간구조·내재상관·SKEW는 **미국 옵션시장 기반이라 KOSPI엔 직접
대응물이 없다**(VKOSPI조차 근사 — v2 §11-4). 따라서:

> **프록시 차용 원칙**: KOSPI에서 직접 계산 불가능한 선행지표는 **S&P500/NASDAQ 지표를 "글로벌 위험
> 프록시"로 그대로 차용**해 KOSPI 대시보드에도 띄운다. KOSPI 단독 신호를 기다리는 것보다 미국 구조
> 경보가 더 빠르고 신뢰성 있는 경우가 많다. UI엔 **"글로벌 프록시(US 기반)"** 라벨을 명시해, 이게
> KOSPI 자체 데이터가 아니라 차용임을 분명히 한다.

지표별 차용 여부:

| 지표 | KOSPI 자체 가능? | 차용 전략 |
|---|---|---|
| VIX 기간구조 역전 | ✗ (VKOSPI 근사, 기간구조 없음) | **US VIX9D/VIX3M를 글로벌 프록시로 KOSPI에도 표시** |
| 내재상관 급등 | ✗ (옵션 데이터 인증벽) | **US implied correlation을 프록시로** + KOSPI는 실현상관 직접계산 보조 |
| 시장 내부 다이버전스 | △ (KRX 인증 시 가능) | KOSPI 자체 계산이 이상적 — 인증이 관문(§14-7-C). 인증 전엔 US breadth 프록시 |
| HY 스프레드/크로스에셋 | ○ (FRED 글로벌 공통) | 차용 불필요 — 원래 글로벌 지표, 양시장 공통 |
| 모멘텀 다이버전스/ROC | ○ (지수 자체) | 각 시장 지수로 직접 계산, 추가 데이터 0 |

## 14-1. 모듈 A — 변동성 기간구조 (1순위, S&P 전환 최강 선행지표)

### 개념

VIX **레벨**(v2에 이미 있음)이 아니라 VIX **구조**를 본다. 평상시 단기 변동성 < 장기 변동성(콘탱고).
이게 **역전(백워데이션)** 되면 = 시장이 "지금 당장의 위험이 먼 미래보다 크다"고 가격화 시작.
큰손이 당장의 헤지를 급히 사들이면 단기 변동성 가격이 먼저 튄다 → **지수가 멀쩡할 때 옵션시장이 먼저
움직이는** 몇 안 되는 진짜 선행신호. **급락 1~2일 전, 때로는 장중에** 선행.

### 계산

```
ts_ratio_short = VIX9D / VIX        // 초단기 vs 30일
ts_ratio_main  = VIX   / VIX3M      // 30일 vs 3개월
역전 판정: ratio > 1.0 (백워데이션 = 경보)
경고대(config): 0.95~1.0 = 평탄화 주의, >1.0 = 역전 점등
SKEW(보조): 외가격 풋 수요 = 테일헤지. 급등 시 "스마트머니 선제 헤지" 보조신호
VVIX(보조): VIX의 변동성. VIX 낮아도 VVIX 급등이면 변동성 폭발 가격화 중
```

### 데이터

- **VIX9D(`^VIX9D`) / VIX(`^VIX`) / VIX3M(`^VIX3M`)** — yfinance, **인증 불필요**. SKEW=`^SKEW`, VVIX=`^VVIX`.
- 일부 티커가 yfinance에서 불안정할 수 있음 → CBOE 공개 데이터 폴백 경로 확인(§14-7-A).

### 적용

S&P/NASDAQ 메인 신호. **KOSPI엔 "글로벌 프록시"로 동일 게이지를 차용 표시**(전제 3).

## 14-2. 모듈 B — 동반붕괴 사전감지 (내재상관 + 순환매 붕괴)

### 개념 — "순환매가 돌다가 일시에 무너진다"의 메커니즘

건강한 상승장은 섹터가 **돌아가며** 오른다(순환매) → 섹터 간 상관 낮음 → 분산이 시장을 받친다.
폭락은 이 구조가 **일시에 깨질 때** 온다: 모든 섹터·종목이 갑자기 같은 방향(아래)으로 동시에 움직임
→ **상관관계가 1로 수렴** → 받쳐줄 게 사라지며 쏠림 매도로 증폭 → 일시 동반붕괴. "순환매가 돌다가
일시에"가 정확히 이 전환이다. **VIX 레벨은 이때 오히려 낮을 수 있어**(그래서 방심) 레벨로는 못 잡고
**구조**로 잡는다.

### 계산

```
[내재상관 - 1순위]
implied_correlation: CBOE 내재상관 지수(가용 시) — 종목간 옵션내재 상관
  → 비정상적으로 낮은 상태(다들 따로 논다)에서 급등 시작 = 동반붕괴 준비 신호
[실현상관 - 폴백/KOSPI 보조]
realized_corr = 구성종목 일별수익률의 평균 페어와이즈 상관(rolling N일)
  또는 근사: 지수변동성 / Σ(개별종목 변동성 가중합)  // 1에 가까울수록 동반화
[분산(dispersion) 보조]
dispersion = 개별종목 변동성 평균 − 지수 변동성  // 압축 시 쏠림·동반붕괴 전조
```

판정: 상관이 **저점에서 상승 변곡**(저상관→급등)할 때 경보. 절대수준보다 **변곡**이 신호.

### 데이터

- US implied correlation: **CBOE 지수(가용성 변동 이력 있음 → §14-7-B에서 먼저 검증)**. 안 되면 실현상관 직접 계산.
- 실현상관: 섹터/종목 시계열(v2 §6에서 이미 수집하는 데이터 재사용) → **추가 데이터 0**.
- KOSPI: 옵션 내재상관 인증벽 → **실현상관 직접계산 + US 내재상관 프록시 병행**.

## 14-3. 모듈 C — 시장 내부 다이버전스 (폭의 균열)

### 개념

지수는 신고가인데 **그 밑에서 폭(breadth)이 먼저 좁아지는** 것. 상승이 소수 대형주에 업혀가는 천장 전조.
v2 §7 워치리스트에 "breadth 신고가 다이버전스 — 수동확인 필요"로 남겨둔 항목을 **자동화하는 게 §14의
핵심 과제**(v2 §7 차기과제로 명시했던 바로 그것).

### 계산

```
net_new_highs = 52주 신고가 종목수 − 52주 신저가 종목수
  다이버전스: 지수 신고가 & net_new_highs 하락추세 = 약세
ad_line = Σ(상승종목수 − 하락종목수)  // 등락주선 누적
  다이버전스: 지수 신고가 & ad_line 신고가 미달 = 약세
mcclellan_osc = EMA19(adv−dec) − EMA39(adv−dec)  // breadth 모멘텀
  0선 하향교차 / 다이버전스 = 경보
pct_above_200ma = 200일선 위 종목 비율
  지수는 200MA 위 & 비율 하락(예: 60%→45%) = 소수 대형주 쏠림
up_down_volume(보조): 상승거래량/하락거래량 — 분산매도(distribution) 탐지
```

### 데이터 — **KRX 인증이 관문(전제 3 표 참고)**

- US: yfinance 구성종목 + 기존 섹터 데이터로 계산 가능. NYSE A-D·신고저는 공개 지표 활용.
- **KOSPI: 종목별 신고저·등락은 pykrx 인증 필요(v2 §11-1).** 인증 전엔 **US breadth를 프록시로** 표시하고
  "KOSPI 자체 breadth 미가용(인증 필요)" 명시. **인증 확보가 KOSPI 전환탐지의 분기점**(§14-7-C).

## 14-4. 모듈 D — 크로스에셋 리스크오프 (가성비 최고, 데이터 이미 보유)

### 개념

주식시장 **밖**(채권·통화·신용)이 위험선호 전환을 먼저 보여준다. 채권·통화시장이 주식보다 빠른 경우가
많다. **대부분 FRED/yfinance로 인증 없이 잡히고 KOSPI·S&P 양쪽 공통**(원래 글로벌 지표) → 차용 불필요.

### 계산

```
hy_oas_roc = HY OAS 스프레드의 N일 변화율(레벨 아닌 "변화 속도")
  저점에서 벌어지기 시작하는 변곡이 주식보다 선행 (v2는 레벨만 봄 — §14가 ROC 추가)
copper_gold = 구리/금 비율  // 글로벌 경기·위험선호 실물 프록시, 주식천장서 선행 하락
dollar_safehaven = DXY + (엔/스위스프랑 강세)  // 안전통화 회피 = 위험회피 선행
  KOSPI는 USD/KRW 급변이 거의 동행경보(v2 S05 극민감)
move_index(보조): 미국채 변동성 — 주식변동성에 선행하는 경우 잦음
```

### 데이터

- HY OAS=FRED `BAMLH0A0HYM2`(v2 이미 보유, ROC만 추가), DXY=`DTWEXBGS`/`DXY`, 구리/금=yfinance,
  엔/스위스=yfinance, MOVE=가용성 확인. **추가 인증 거의 없음 → 가성비 최고, 우선 착수 권장.**

## 14-5. 모듈 E — 모멘텀 다이버전스 + 가속도 둔화 (추가 데이터 0)

### 개념

지수 자체로만 계산. v2/§13의 밴드·이격도를 넘어 **모멘텀의 약세 다이버전스와 2차 미분(가속도)** 을 본다.

### 계산

```
rsi_divergence: 가격 신고가 & RSI 저점 낮아짐(lower high) = 약세 다이버전스
macd_hist_divergence: 가격 신고가 & MACD 히스토그램 약화 = 모멘텀 소진
roc_accel = ROC(t) − ROC(t-k)  // 상승의 2차 미분
  오르긴 하나 "오르는 속도"가 둔화 = 천장 근접 (§13 RRG momentum과 같은 발상, 지수에 적용)
atr_expansion(보조): 추세후반 일중변동폭 확대 = 전환 임박
```

- 전부 지수 OHLCV로 계산 → **추가 데이터 0**. config로 윈도우 전부 노출.

## 14-6. 통합 — 전환 경보 합의 게이지 (Regime Shift Alert)

### 핵심 구조 — confluence(전제 2)

개별 모듈이 아니라 **합산 점수**로 경보한다. 각 모듈을 −2(강한 전환신호) ~ +2(강한 상승지속)로 정규화:

```
modules = { A: vix_term, B: cocrash, C: internals_div, D: crossasset_riskoff, E: momentum_div }
각 모듈 → score ∈ {-2,-1,0,+1,+2}
alert_score = Σ(wᵢ · scoreᵢ)   // 음수일수록 전환위험

경보 단계(config):
  점등 모듈 수 < 3        → 정상 (개별 신호는 노이즈로 간주, 경보 억제)
  점등 모듈 수 ≥ 3        → 🟡 주의 (방어자세 전환 검토)
  점등 ≥ 3 & A(VIX역전) 포함 & B(상관급등) 포함 → 🔴 동반붕괴 경계
```

> **"3개 이상 동시 점등할 때만"** 이 핵심. 단일 모듈 점등은 의도적으로 무시한다 — 오탐을 견딜 수준으로
> 낮추는 유일한 방법. VIX역전+내재상관이 함께 켜지는 게 §14-2의 동반붕괴 시그니처.

### 화면 배치 — 신규 최상단 경보 띠

크로스 내러티브 배지(v2 §3 화면 최상단) **바로 아래** 또는 그와 통합해, 경보가 켜질 때만 눈에 띄게:

```
┌─────────────────────────────────────────────────────────────┐
│ 흐름▸여력▸주도▸정합성 — 크로스 내러티브 배지 (v2)            │
├─────────────────────────────────────────────────────────────┤
│ ⚠ 전환경보: 🟡 주의 (4/5 모듈 점등)  [글로벌 프록시 포함]    │ ← §14 신규
│  A.VIX역전 ● │ B.상관급등 ● │ C.breadth약세 ● │ D.리스크오프 ●│ E.모멘텀 ○│
│  "겉은 신고가, 속은 쏠림+옵션헤지+분산소멸 — 방어 검토 구간"  │

```

## 14-9. 구현 시 반드시 알아야 할 것 (§14 하드 러슨)

### 14-9-A. VIX 보조 티커는 yfinance에서 불안정할 수 있다

`^VIX`는 안정적이나 `^VIX9D`/`^VIX3M`/`^SKEW`/`^VVIX`는 yfinance에서 누락·지연되는 경우가 보고된다.
**CBOE 공개 데이터 폴백 경로를 미리 확보**하고, 둘 다 실패 시 모듈 A를 "미가용"으로 degrade(silent
오류 금지 — v2 §3-C 원칙). 메인 신호 ^VIX/^VIX3M부터 검증하고 9D는 보조로 둔다.

### 14-9-B. 내재상관 지수(CBOE)는 상품 변경 이력이 있어 가용성을 먼저 확인해야 한다

CBOE implied correlation 지수는 과거 코드·산출방식이 바뀐 이력이 있다. **9d 착수 첫 작업은 "현재
무료로 받을 수 있는 내재상관 시리즈가 있는가" 검증**이다. 없으면 곧장 **실현상관 직접계산**(섹터/종목
시계열은 v2 §6에서 이미 수집 — 추가 데이터 0)으로 가고, "실현상관 근사"로 라벨링한다(v2 §11-4 방식).

### 14-9-C. KOSPI 시장내부(모듈 C)는 KRX 인증이 분기점 — 단순 편의가 아니다

Net New Highs·A-D선·%above200MA는 전부 종목별 데이터가 필요하고 KOSPI는 pykrx 인증(v2 §11-1)에
막힌다. **이건 "있으면 좋은" 게 아니라 KOSPI 전환탐지를 하느냐 마느냐의 분기점.** 인증 전엔 US
breadth를 글로벌 프록시로 띄우되, "이건 KOSPI 자체 폭이 아니라 미국 폭"임을 UI에 분명히 한다.
KOSPI 자체 breadth 다이버전스의 선행성은 인증 확보 후에야 검증 가능.

### 14-9-D. 기간구조 역전은 오탐이 적중보다 많다 — "예측"이 아니라 "확률 상승 구간"

역전됐다가 폭락 없이 며칠 만에 정상화되는 경우가 적중보다 흔하다. 그리고 2018-02 볼마게돈·2020-03
처럼 **변동성 자체가 폭발 원인**인 경우(VIX 연동상품 청산이 시장을 끌어내림)는 역전이 거의 동시에
일어나 "사전" 여유가 없다. §14는 "확실한 예측 도구"가 아니라 **"방어자세 전환 확률이 평소보다 유의미
하게 높아진 구간"** 표시기다. UI 문구·면책에 이 한계를 반드시 명시한다.

### 14-9-E. confluence 최소 모듈 수를 낮추면 경보가 노이즈가 된다

`alert_min_modules`를 2 이하로 내리고 싶은 유혹이 크지만(더 빨리 잡고 싶어서), 그러면 오탐이 폭증해
경보를 아무도 안 믿게 된다. **3 이상 유지**가 설계의 핵심. 더 민감하게 하려면 모듈 수를 늘려 분모를
키우는 방향으로 가지, 임계를 내리지 않는다.

### 14-9-F. "글로벌 프록시"를 KOSPI 자체 신호처럼 보이게 하면 사용자가 오판한다

US VIX 구조·US breadth를 KOSPI 화면에 차용 표시할 때, 출처 라벨이 없으면 사용자는 "KOSPI가 이렇다"로
오해한다. **모든 차용 모듈에 "글로벌 프록시(US)" 뱃지를 강제**한다. KOSPI는 미국에 후행하므로 프록시가
유효하지만, "후행"이라는 시간차(보통 반나절~하루)도 함께 안내해 KOSPI에 그대로 1:1 적용하지 않게 한다.

### 14-9-G. 백테스트 없는 전환경보는 작동 안 하는 장식이다 — 9f를 건너뛰지 말 것

가장 중요한 하드 러슨. 그럴듯한 선행지표를 모아 게이지를 만드는 것까진 쉽지만, **실제로 선행성이
있는지는 백테스트로만 안다.** 9f에서 "경보 점등일 기준 +1/+3/+5/+10일 지수수익률 분포"를 산출해
경보 없는 날 분포와 비교(평균·하방꼬리·승률)하고, 선행성이 확인된 모듈에 가중치를 싣는다. 선행성이
안 나오는 모듈은 가중치를 빼거나 보조로 강등한다. **이 검증 전엔 경보를 사용자에게 노출하지 않는다.**

## 부록 — §14 면책 (v2/§13과 동일 기조, 더 강한 경고)

전환 경보·동반붕괴 사전감지는 **확실한 예측이 아니다.** 모든 모듈은 오탐을 내포하며, 특히 변동성
기간구조 역전은 폭락 없이 정상화되는 경우가 적중보다 흔하다(§14-9-D). 이 레이어는 "방어 자세 전환을
검토할 확률이 평소보다 유의미하게 높아진 구간"을 표시할 뿐이며, 매매 시점을 단정하지 않는다. KOSPI에
표시되는 글로벌 프록시 신호는 미국 시장 기반 차용으로, 후행 시간차가 존재한다. 백테스트로 선행성이
검증되지 않은 모듈은 노출하지 않는다. 투자 판단과 책임은 사용자 본인에게 있다.
