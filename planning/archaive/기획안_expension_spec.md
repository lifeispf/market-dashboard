# 유동성 천장 모니터링 대시보드 — 확장 스펙 §13 (시장 방향성 레이어)

> 이 문서는 v2 기획안(`기획안.md`)의 **추가 분책(§13)** 이다. v2의 4단 모델(흐름▸여력▸주도▸정합성)은
> "유동성이 받칠 수 있는 천장이 어디인가"를 본다. 그러나 v2를 검토하면서 **"증시 자체의 방향성"**
> — 지금 시장이 위로 갈 환경인가 아래로 갈 환경인가 — 을 직접 묻는 레이어가 빠져 있다는 게 드러났다.
> §13은 그 방향성을 세 개의 독립 신호로 보강한다. **v2 본문(§1~§12)을 먼저 읽은 사람을 전제**로 쓴다.
> 용어·데이터소스 표기·config 분리 원칙·"하드 러슨" 컨벤션은 전부 v2를 그대로 따른다.

---

## 13-0. 왜 이 세 가지인가 — 한 줄 철학

v2가 답하는 건 "**얼마나 갈 수 있나**(여력)"와 "**어디로 가고 있나**(주도)"다.
§13이 더하는 건 "**지금 환경이 어느 쪽으로 기울어 있나**(방향성)"다. 세 신호는 서로 다른 시간축·다른
참여자 심리를 본다:

| 확장 | 질문 | 시간축 | 본질 |
|---|---|---|---|
| **① 공포·탐욕 지수** | 참여자들이 지금 얼마나 흥분/패닉 상태인가 | 단기(일~주) | **심리·포지셔닝** |
| **② 가격 위치(밴드)** | 가격이 자기 레인지·기술적 채널 안 어디에 있나 | 단·중기 | **기술적 위치** |
| **③ 섹터 공·수 로테이션** | 돈이 공격 섹터로 가나 방어 섹터로 가나 | 중기(주~월) | **자금 회전 방향** |

**핵심 분리 원칙**: 이 셋은 v2의 REGIME 합성점수(S01~S06)에 **절대 섞지 않는다.** REGIME은 "객관적
유동성 연료량"이고, §13은 "심리·기술·자금흐름의 방향"이다. 둘이 어긋날 때(예: 연료는 차 있는데 심리는
극단적 탐욕) 바로 그 **불일치 자체가 새로운 신호**이므로, 합산해서 평균 내버리면 정보가 사라진다.
별도 게이지로 나란히 두고 **교차 해석**한다.

## 13-1. 공포·탐욕 지수 (Fear & Greed) — 독립 게이지

### 개념

CNN Fear & Greed Index 방식의 다요인 합성. **여러 시장 내부 지표를 0–100으로 정규화 → 평균 →
5단계 라벨.** v2 §4의 REGIME과 형태(0–100 게이지)는 같지만 **입력도 다르고 의미도 다르다** — 화면에서
색·위치를 분명히 구분해 "이건 연료, 저건 심리"가 헷갈리지 않게 한다.

### 팩터 구성

| 팩터 | 계산 | 데이터 상태 | 0=공포 / 100=탐욕 방향 |
|---|---|---|---|
| **F1. 모멘텀** | 현재 지수 ÷ 125일 단순이평 − 1 | **이미 있음**(지수 OHLCV) | 이평 위 = 탐욕 |
| **F2. 강도(breadth)** | 상승종목수 ÷ (상승+하락) | **이미 있음**(KOSPI breadth) / NASDAQ는 상승섹터수 대용 | 상승 우세 = 탐욕 |
| **F3. 변동성** | VIX / VKOSPI(근사)의 역방향 정규화 | **이미 있음** | 저변동 = 탐욕 |
| **F4. 신용시장** | HY OAS 스프레드의 역방향 정규화 | **이미 있음**(FRED BAMLH0A0HYM2) | 타이트 = 탐욕 |
| F5. 안전자산 선호 | 주식 20일수익률 − 안전자산(국고채/TLT류) 20일수익률 | 신규(국고채/장기채 시계열 필요) | 주식 우위 = 탐욕 |
| F6. 풋·콜 비율 | 옵션 P/C ratio의 역방향 | 신규·**막힐 가능성 높음**(§13-7-A) | 콜 우위 = 탐욕 |

### 단계별 구현 — "근사부터, 확장은 나중에"

VKOSPI를 "근사"로 처리한 v2 §11-4의 철학을 그대로 따른다. **F5·F6은 데이터벽에 막힐 수 있으니
처음부터 의존하지 않는다.**

- **MVP (4팩터)**: F1·F2·F3·F4 — 전부 v2가 이미 가진 데이터. 인증 불필요. UI에 **"4팩터 근사"** 명시.
- **확장 (5~6팩터)**: F5(안전자산), F6(풋콜)을 데이터 확보 후 추가. 추가 시 가중치 재정규화.

> **REGIME과 동일한 재정규화 원칙**(v2 §3-C): 일부 팩터가 미가용이면 **available한 팩터만으로 가중치를
> 재정규화**하고 "반영팩터 N/6"을 같이 표시한다. silent하게 틀린 점수를 보여주면 안 된다.

### 스코어링

```
각 팩터 → 0–100 정규화 (방향 통일: 항상 0=공포, 100=탐욕)
fear_greed = Σ(wᵢ · factorᵢ) / Σ(wᵢ, available only)

기본 가중치(config): F1 .25 · F2 .20 · F3 .20 · F4 .15 · F5 .10 · F6 .10
  → MVP(F1~F4)에서는 .25/.20/.20/.15를 합 .80으로 나눠 재정규화

5단계 라벨(config로 컷 조정 가능):
   0–24  Extreme Fear   (극단적 공포)
  25–44  Fear           (공포)
  45–55  Neutral        (중립)
  56–74  Greed          (탐욕)
  75–100 Extreme Greed  (극단적 탐욕)
```

### 정규화 앵커 (config — 임의 자리값, 서비스화 전 검토 필요)

> v2 §5-1이 S03/S04/S06 임계를 "임의 자리값"으로 명시한 것과 같은 취급. 아래는 출발점일 뿐이다.

```jsonc
"fear_greed_anchors": {
  "F1_momentum":  { "lo_pct": -8,  "hi_pct": 8 },      // 이평 대비 -8%→0, +8%→100
  "F2_breadth":   { "lo": 0.30, "hi": 0.70 },          // 상승비율 30%→0, 70%→100
  "F3_vol":       { "lo": 12,   "hi": 30, "invert": true }, // VIX 12→100, 30→0
  "F4_oas":       { "lo_pct": 3, "hi_pct": 6, "invert": true }, // OAS 3%→100, 6%→0
  "F5_safehaven": { "lo_pct": -10, "hi_pct": 10 },     // 주식-채권 20d수익률차
  "F6_putcall":   { "lo": 0.7,  "hi": 1.3, "invert": true }
}
```

### 화면 배치

v2 §3 레이아웃의 **C(REGIME 게이지) 바로 옆에 짝으로** 배치한다. 두 게이지를 나란히 두는 게 핵심:

```
├──────────────────────────────┬──────────────────────────────┤
│ C. 유동성 REGIME 게이지       │ C2. 공포·탐욕 게이지 (신규)   │
│   ░░░░░░░▓▓ 44/100           │   ░░░░▓▓▓▓ 68/100            │
│   "BULL 연료 부분충전"        │   "Greed · 탐욕(4팩터 근사)"  │
│   반영원천 6/6                │   반영팩터 4/6                │
├──────────────────────────────┴──────────────────────────────┤
```

### 교차 해석 → 정합성 배지에 4번째 케이스 추가

v2 §4-3의 정합성 배지는 가격 vs 유동성만 비교했다(🟢정합/🔴과열/🟡여유). §13은 여기에 **심리 오버레이**를
한 줄 추가한다. 별도 상태가 아니라 기존 배지에 붙는 **보조 경고 텍스트**:

| REGIME(연료) | 공탐(심리) | 보조 해석 |
|---|---|---|
| 충전 중(높음) | 극단적 탐욕(높음) | ⚠ 연료는 있으나 단기 과열 — 진입은 신중 |
| 마름(낮음) | 극단적 공포(낮음) | 🟢 연료는 적으나 심리 바닥 — 반등 가능 구간 |
| 충전 중 | 공포/중립 | 🟢 연료 있고 심리 미달궈짐 — 가장 우호적 조합 |
| 마름 | 탐욕 | 🔴 연료 없는 탐욕 — 가장 취약한 조합 |

## 13-2. 가격 위치(밴드) — 세 가지 기술적 위치 지표

### 개념

v2 §3-B의 "천장 밴드 차트"는 **유동성이 미래에 지지할 천장**(forward-looking, 밸류에이션 기반)이다.
§13-2는 그 차트의 짝으로 **가격이 자기 자신의 최근 궤적 안 어디에 있나**(backward-looking, 가격 기반)를
본다. 세 지표 전부 **추가 데이터 0개** — 이미 가진 지수 OHLCV만으로 계산된다.

### 13-2-A. 단순 가격 레인지 (52주 위치)

```
range_pct = (현재가 − 52주최저) / (52주최고 − 52주최저) × 100
거리지표: 52주최고 대비 %, 52주최저 대비 %
```

- 0%=연저점, 100%=연고점. "현재 9,064 → 52주 레인지 상위 87% · 전고점 -3.2%" 식으로 표시.
- **config**: `range_window_days`(기본 252거래일). 5년 레인지도 보고 싶으면 멀티 윈도우(252/1260) 토글.

### 13-2-B. 볼린저 밴드

```
mid   = N일 단순이평 (기본 20)
upper = mid + k × N일 표준편차 (기본 k=2)
lower = mid − k × N일 표준편차
%B    = (현재가 − lower) / (upper − lower)   // 밴드 내 위치 0~1
bandwidth = (upper − lower) / mid             // 밴드 폭(변동성 압축/확장)
```

- **%B**: 1 초과=상단 돌파(과매수), 0 미만=하단 이탈(과매도). 화면엔 0~100%로.
- **bandwidth**: 좁아지면(스퀴즈) 변동성 확대 임박 신호 → 워치리스트 트리거로도 활용.
- **config**: `bollinger`: `{ "window": 20, "k": 2 }`.

### 13-2-C. 이동평균 이격도 (Disparity)

```
disparity[n] = 현재가 / n일이평 × 100   // 100=이평선상, >100=위, <100=아래
기본 추적: 20일·60일·120일 (config로 조정)
```

- 100 기준. 단기(20)·중기(60)·장기(120) 이격도를 동시에 보면 추세 정렬/괴리 판단.
- 단기 이격도가 과도하면(예: >110) 단기 과열 → 평균회귀 압력. 워치리스트 연계.
- **config**: `disparity_windows`: `[20, 60, 120]`.

### 화면 배치

v2 §3 레이아웃의 **B(천장 밴드 차트) 바로 아래**에 가로 게이지 3종 스택:

```
├──────────────────────────────┬──────────────────────────────┤
│ B. 천장 밴드 차트 (forward)   │ C. REGIME · C2. 공탐 게이지   │
├──────────────────────────────┴──────────────────────────────┤
│ B2. 가격 위치 (backward, 신규)                               │
│   52주 레인지  ▕━━━━━━━━●──▏ 87%ile · 전고점 -3.2%           │
│   볼린저 %B    ▕━━━━━●─────▏ 62% · 폭 정상                    │
│   이격도       20d:+4.1% 60d:+9.8% 120d:+18.2% (정렬·상방)   │
├──────────────────────────────────────────────────────────────┤
```

> **B(forward)와 B2(backward)를 위아래로 붙이는 이유**: "유동성은 아직 위를 가리키는데(B) 가격은 이미
> 자기 레인지 천장에 닿았다(B2)" 같은 조합이 한눈에 잡힌다. 정합성 배지의 보조 근거가 된다.

## 13-3. 섹터 공·수 로테이션 — Layer 3(주도) 보강

### 개념

v2 §3의 RRG는 "**어느 섹터가 1등이냐**"(상대강도×모멘텀)는 보여주지만 "**지금이 공격 국면이냐 방어
국면이냐**"는 안 보여준다. 방어주(통신·필수소비·유틸리티)에서 경기민감주(반도체·산업재·임의소비)로 돈이
이동하면 **위험선호 확대(상승 환경)**, 반대면 **방어 회전(후반·조정 환경)**. 이게 v2에 빠진 방향성 신호다.

### 스키마 — sectors.json에 필드 하나 추가

v2 §6의 수동 큐레이션 철학을 그대로 따른다. **계산되는 값이 아니라 사람이 판단해 분류하는 값**이므로
`sectors.json`에 섹터별 `cycle_role` 한 필드만 추가한다:

```jsonc
"semi": {
  "name": "반도체",
  "tickers": [...],
  "cycle_role": "offensive",   // ← 신규. "offensive" | "defensive" | "neutral"
  ...
}
```

### 기본 분류 시안 (검토 후 sectors.json에서 자유롭게 수정)

> v2 §11-5처럼 **LLM 일반지식 기반 분류**이며 검증 안 됨. 특히 "중립"으로 둔 섹터는 팀 합의 필요.

**KOSPI**

| 섹터 | cycle_role | 근거 |
|---|---|---|
| 반도체 | offensive | 경기·실적 사이클 민감, 고베타 |
| 전력기기 | offensive | 캐펙스 사이클 연동 |
| 증권 | offensive | 거래대금·위험선호 직결 |
| 자동차 | offensive | 경기소비재 |
| 화학·2차전지 | offensive | 경기·소재 사이클 |
| 조선·기계 | offensive | 산업재 사이클 |
| 은행·지주 | **neutral** | 고베타지만 배당 방어 성격 혼재 — **대표적 판단 필요 케이스** |
| 바이오 | **neutral** | 성장(공격)과 방어 성격 혼재 |
| 통신 | defensive | 배당·저베타·필수재 |
| 유통·필수소비 | defensive | 필수소비, 경기 비민감 |

**NASDAQ** (GICS ETF 기준 — 매핑 자체는 자동)

| ETF | 섹터 | cycle_role |
|---|---|---|
| XLK / XLY / XLI / XLB / XLF / XLC | Tech·임의소비·산업재·소재·금융·커뮤니케이션 | offensive |
| XLP / XLU / XLV | 필수소비·유틸리티·헬스케어 | defensive |
| XLE | 에너지 | neutral(원자재 사이클 별개) |
| XLRE | 리츠 | neutral(금리민감) |

### 스코어링 — 공·수 스프레드

```
offensive_basket = cycle_role=="offensive" 섹터들의 평균 모멘텀(예: 20일 수익률 또는 RS-Ratio)
defensive_basket = cycle_role=="defensive" 섹터들의 평균 모멘텀
rotation_spread  = offensive_basket − defensive_basket   // >0 위험선호, <0 방어회전

추세: rotation_spread의 N일 변화(기본 20일) 부호
  스프레드 상승 추세 → "공격 우위 · 위험선호 확대"
  스프레드 하락 추세 → "방어 우위 · 위험회피 진입"
  0 근처 횡보       → "중립 · 방향성 불명확"
```

- **모멘텀 정의는 config로**: `rotation`: `{ "momentum_window": 20, "metric": "return" | "rs_ratio" }`.
- neutral 섹터는 바스켓에서 **제외**(방향성을 흐리지 않기 위해). 단 트리맵·RRG에는 그대로 표시.

### 화면 배치

v2 §3의 **주도(Leadership) 섹션, RRG 옆 또는 아래**에 미니 게이지/추이선 추가:

```
│ 주도(Layer3): 섹터 트리맵 │ RRG 순환매                     │
│                           │ ─────────────────────────────  │
│ 공·수 로테이션 (신규)     │ rotation_spread: +6.2 (상승추세)│
│   공격 ▕━━━━━━━●──▏ 방어  │ "공격 우위 · 위험선호 확대"     │
│   20일 추이 스파크라인 ▁▂▃▅▆▇                              │
```

### 크로스 내러티브 배지 연동

## 13-6. 빌드 단계 (v2 §10 Phase 8로 이어붙임)

> v2 §10은 Phase 1~7. §13은 **Phase 8 단일 묶음**으로, 데이터 난이도 오름차순으로 진행한다.
> 각 단계 끝에 v2와 동일하게 "synthetic 단위검증 → 실데이터 엔드투엔드 → 서버기동 확인" 3단계.

**Phase 8 — 시장 방향성 레이어** (난이도 오름차순)

1. **8a. 섹터 공·수 로테이션** — sectors.json에 `cycle_role` 추가 + 공·수 스프레드 계산
   + 주도 섹션 미니 게이지. *기존 섹터 시계열 재사용 → 추가 데이터 0, 가장 쉬움.*
2. **8b. 가격 위치(밴드)** — 52주 레인지 + 볼린저 + 이격도 → B2 블록.
   *지수 OHLCV만 → 추가 데이터 0.*
3. **8c. 공포·탐욕 4팩터 MVP** — F1~F4(전부 기존 데이터) → C2 게이지 + 정합성 보조 해석.
   *F2(KOSPI breadth)는 v2 §11-1 인증벽 동일 — 인증 없으면 NASDAQ만 우선.*
4. **8d. 공탐 확장 + 크로스 연동** — F5·F6 데이터 확보 시 추가(재정규화) + 크로스 내러티브 배지 보강.
   *F5·F6 데이터벽 검증부터(§13-7) — 막히면 4팩터 근사로 확정하고 "근사" 라벨 유지.*

## 13-7. 구현 시 반드시 알아야 할 것 (§13 하드 러슨 — 선제 경고)

> v2 §11과 같은 형식. 단, §13은 아직 구현 전이므로 "예상되는 함정"이다. 8d 착수 전 먼저 검증할 것.

### 13-7-A. 풋·콜 비율(F6)은 KOSPI에서 막힐 가능성이 매우 높다

KOSPI 옵션 P/C ratio는 KRX 파생상품 데이터로, **v2 §11-1의 pykrx 인증벽과 같은 패턴**일 공산이 크다.
US는 CBOE가 공개하지만 무료·안정 API 여부 확인 필요. **F6은 처음부터 `enabled:false`로 두고, 4~5팩터로
시작**하라. VKOSPI를 "근사"로 처리한 v2 §11-4와 동일한 타협이다.

### 13-7-B. 안전자산 선호(F5)의 KR 안전자산 티커가 비어 있다

US는 TLT(장기국채 ETF)로 깔끔하지만, **KR 국고채를 yfinance로 받을 안정적 티커가 불확실**하다
(`safe_ticker_KR: null`로 둔 이유). KOSEF/KODEX 국고채 ETF(예: `148070.KS`류) 또는 FRED 한국 국채금리
시계열로 대체 검토. **검증 전엔 KOSPI F5도 비활성**, NASDAQ만 5팩터로 가는 게 안전하다.

### 13-7-C. 공탐지수를 REGIME과 같은 게이지로 그리면 사용자가 반드시 혼동한다

둘 다 0–100 게이지라 시각적으로 똑같아 보인다. **색 팔레트를 분리**(REGIME=brass계열 "연료", 공탐=별도
색 "심리")하고, 라벨에 "연료" / "심리"를 명시하라. 13-1 교차해석 표가 작동하려면 둘이 **다른 것**임이
화면에서 즉시 읽혀야 한다.

### 13-7-D. 볼린저 %B와 52주 레인지는 다른 것 — 둘 다 "밴드 내 위치"라 헷갈린다

52주 레인지(13-2-A)는 **절대 고저 기준 위치**, 볼린저 %B(13-2-B)는 **이평±표준편차 동적 밴드 기준
위치**다. 추세장에서 둘이 크게 어긋날 수 있다(가격이 52주 신고가인데 볼린저 상단은 계속 따라 올라가
%B는 중간). **버그가 아니라 정의 차이** — v2 §11-6(RRG momentum)과 같은 "정상 동작인데 의심받는" 케이스.

### 13-7-E. cycle_role "neutral" 섹터를 로테이션 스프레드에 넣으면 신호가 죽는다

은행·바이오·에너지·리츠를 공격/방어 중 하나로 억지 분류하면 스프레드가 흐려진다. **neutral은 스프레드
계산에서 제외하되 트리맵·RRG엔 그대로 표시**(`exclude_neutral:true`). v2 §6의 "테제 없는 종목을 억지로
만들지 않는다"와 같은 절제 원칙.

### 13-7-F. KOSPI breadth(F2)가 인증벽이면 공탐지수 자체가 KOSPI에서 반쪽이 된다

F2(강도)는 KOSPI에서 pykrx breadth에 의존하는데 이건 v2 §11-1 인증 필요다. 인증 없으면 KOSPI 공탐은
F1·F3·F4 **3팩터**로 더 줄어든다. NASDAQ은 섹터 등락종목수로 대용 가능. **KOSPI/NASDAQ의 반영팩터 수가
다를 수 있음을 UI에 명시**("KOSPI 3팩터 / NASDAQ 5팩터" 식)하고 점수를 같은 선상에서 직접 비교하지 말 것.

## 부록 — §13 면책 (v2와 동일 기조)

공포·탐욕 지수, 가격 위치 밴드, 섹터 공·수 로테이션은 모두 **시장 환경을 읽기 위한 정성적·정량적
보조 신호이며 예측이 아니다.** 특히 cycle_role 분류와 정규화 앵커는 사람이 정한 판단값으로,
분기 단위로 재검토해야 한다(v2 §6 `as_of` 철학). 투자 판단과 책임은 사용자 본인에게 있다.
