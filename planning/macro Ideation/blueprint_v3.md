# 유동성 천장 모니터링 대시보드 — 모듈 전체 지도 (Blueprint v3)

> **이 문서가 대체하는 것**: 기존 `blueprint.md`(v1 지도). v1은 §13/§14/§15 세 스펙에 흩어진
> 6개 출력축을 한 장에 모은 *색인*이었다. v3는 그 위에 **실증 검증(이벤트스터디·walk-forward·
> triage·스파이크강도)으로 드러난 "각 모듈의 진짜 역할"과, OpenBB 데이터레이어 채택, 신규 신호축
> 3종, 그리고 검증 게이트라는 아키텍처 원칙**을 반영한 개정 지도다. 상세 계산·config는 `기획안_v3.md`
> 참조. 이 문서는 **무엇이 몇 개 있고, 실제로 무엇을 하며, 서로 어떻게(안) 합쳐지는지**만 본다.

## 0. v1 → v3에서 바뀐 것 (요약)

| 영역 | v1 | v3 |
|---|---|---|
| 출력축 | 6개(여력·주도·공탐·가격위치·로테이션·전환경보) | **6개 유지 + Triage 레이어(7번째) 추가** |
| 모듈 역할 | 설계상 "선행" 가정 | **실증 재분류**: 대부분 동행/후행, 진짜 선행후보는 C·D 중심(§2) |
| 신호축 | 가격·변동성·유동성·심리 | **+ 포지셔닝(FINRA 공매도·다크풀)·정책캘린더·풋콜비율** |
| 데이터 | 손수 짠 fetch(CBOE cdn·FRED CSV) | **OpenBB 데이터레이어**로 추상화(키 1개=FRED만)(§5) |
| 산출 형태 | 게이지·경보(이진) | **확률(0~1)·base-rate 보정·확률상한 0.6**(§4) |
| 배포 기준 | 없음 | **검증 게이트**: 백테스트 통과 전 방향·강도·확률 비노출(§4) |

핵심 철학은 그대로다 — **6축은 절대 합산하지 않고, 어긋남(불일치)이 가장 큰 신호다.** v3는 이 원칙을
버리지 않고, "각 축이 실제로 무슨 일을 하는지"를 데이터로 못 박아 *과신을 제거*하는 방향의 개정이다.

## 2. 모듈 역할 재분류 — 설계 주장 vs 실증 (v3의 핵심)

walk-forward(실측 SPX+VIX 1990–2019, 기저율 P(20일−8%)=6.4%)와 이벤트스터디가 드러낸 것:
**"선행이면서 예측적인" 신호는 거의 없다.** 예측적인 건 후행이고, 선행위치에서 켜지는 건 반예측적.

| 모듈/서브 | 설계상 주장 | **실증 결과** | v3 처분 |
|---|---|---|---|
| §14-A VIX 기간구조 | "1순위 최강 선행" | VIX레벨은 정밀도 있으나 **후행**(점등 시 23%만 고점권), 오탐多 | **강등**: 보조·확인 + X5 구성요소 |
| §14-B 내재/실현상관 | 동반붕괴 선행 | 상관은 폭락 *때* 1로 수렴 = **순수 동행** | **X5 veto 전용**, 단독가중↓ |
| §14-C breadth | 천장 전조 | 느린 분산천장에서 **수주~수개월 선행**(2021·2026·2023) | **승격**: 천장 1순위, T2 가중↑ |
| §14-D 크로스에셋 | 가성비 | 신용·금리·통화 트리거 **범용 선행**, 빠른장(SVB·엔캐리)도 유일하게 선행 | **승격**: 최우선 착수 |
| §14-E 모멘텀 | 추가데이터0 | C와 짝으로 선행 | 유지·T2 가중↑ |
| 공탐·가격위치(극단) | 과열=약세 | **무조건부로는 반예측적**(리프트<1) — 추세장에선 지속 | **레짐 조건부 필수**(§15-6-D) |
| REGIME | 연료 | 타이밍은 못 잡고 **취약성 맥락**(모든 천장이 고밸류서 발생) | T3 가중↑, T1=0 유지 |

**§14 내부 가중 재틸트(prior, walk-forward 보정 전)**: A .25→.18 · B .25→.12 · C .20→.25 · D .15→.23 · E .15→.22.

> 결론: 가격·VIX만으로는 루브릭이 **선행 못 하고 "확인"만 한다.** 진짜 선행력은 C(breadth)·D(크로스에셋)
> 데이터 인프라 확보에 달려 있다 — 그게 §5 OpenBB 채택의 직접 동기다.

## 3. 신규 신호축 3종 — 기존 모듈에 편입 (새 게이지를 늘리지 않음)

OpenBB 카탈로그 점검에서 나온, 기존 프레임의 **공백을 메우는** 축들. 별도 7·8번째 게이지로 늘리지
않고 기존 모듈을 보강한다(합산 금지 원칙 유지).

| 신규 축 | 데이터(OpenBB) | 어디에 편입 | 메우는 공백 |
|---|---|---|---|
| **정책 캘린더** (FOMC·CPI 일정) | `economy.calendar` | **Triage 피처 + 정합성 배지** | 이벤트스터디 최대 공백 — 바닥 6/6의 진짜 방아쇠가 정책촉매였으나 어떤 모듈도 못 잡음. §15-7-E의 수동 `policy_pivot_watch`를 **자동화** |
| **풋콜 비율** | `derivatives.options.chains`(cboe, 키X) | **공탐 F6 부활** | §13-7-A가 인증벽 우려로 `enabled:false` 했던 F6을 US 체인 직접계산으로 복구 |
| **포지셔닝**(공매도·다크풀) | `equity.shorts.*`·`equity.darkpool.otc`(finra/stockgrid, 키X) | **Triage 피처 + 전환경보 보조** | 프레임에 없던 축. 공매도급증+급락=항복, 급락후 숏커버=데드캣 구별 |

(보조: `money_measures`(M1/M2)·`primary_dealer_positioning` → REGIME 유동성측 보강, `futures.curve('VX')` → 모듈 A 기간구조 정밀화.)

## 4. 신규 아키텍처 원칙 3·4·5 (1·2는 v1 계승)

1. **(계승) 6축 합산 금지.** 평균 내면 어긋남 정보가 사라진다.
2. **(계승) 불일치가 신호.** §15 충돌 카탈로그(X1~X7)로 형식화.
3. **🆕 검증 게이트.** 방향·강도·확률은 walk-forward 백테스트(base rate·정밀도·calibration) 통과 전
   **사용자에게 노출 금지**(`expose_only_if_backtested`). 검증 안 된 산출은 "장식"(§14-9-G).
4. **🆕 확률 규율.** 모든 확률은 **base-rate 인지**(6.4% 기저 위에서) + **상한 0.6**. 가장 좋은 조합도
   실측 정밀도 22%·리프트 3.5배에 불과 → 0.7+ 확률 금지.
5. **🆕 선행/동행 정직성.** 대부분 신호는 *확인*이지 예측이 아니다. 현실적 용도는 **Triage(사이징 다이얼)**:
   "지금 빠지는 dip이 깊어질 확률"을 연속 출력해 포지션·헤지 강도를 조절. 단건 예측 도구로 팔지 않는다.

## 5. 데이터 레이어 — OpenBB 채택 + 갭

손수 짠 CBOE cdn 호출·FRED CSV 파싱을 `obb.*`로 추상화. **무료 키는 FRED 하나**, 나머지 키리스.

| 모듈 | OpenBB 경로 | 키 | v3 상태 |
|---|---|---|---|
| A 기간구조 + **NASDAQ** | `index.price.historical(['VIX','VIX9D','VIX3M','VVIX','SKEW','^IXIC'], provider='cboe')` | 불필요 | ✅ 해결(NASDAQ 차트 문제 포함) |
| D HY OAS | `fixedincome.corporate.ice_bofa(index_type='spread', grade='high_yield')` | **FRED(무료)** | ✅ 손파싱 제거 |
| D 구리/금·DXY·JPY | `commodity.*` · `currency.*` | 불필요 | ✅ |
| C breadth | 구성종목 루프 + `equity.price.historical(provider='yfinance')` | 불필요 | ⚠️ **여전히 직접 계산**(키리스 시계열 엔드포인트 없음) |
| 정책캘린더·풋콜·포지셔닝 | §3 표 | 불필요 | 🆕 |
| KOSPI breadth | (pykrx 인증) | **인증벽** | ❌ 미해결 — US 글로벌 프록시 유지(§14 전제3) |

> OpenBB는 AGPL v3 — 상업 임베딩 시 코드공개 의무 주의. 자체 호스팅·연구용은 무관.
